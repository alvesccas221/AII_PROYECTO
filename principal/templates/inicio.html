En esta página se muestra una descripción del sistema y un resumen de lo que hace cada una de las pestañas disponibles.

Lista de juegos almacenados en BD

En la pestaña "Juegos" podremos observar un listado de títulos de juegos de mesa. Estos títulos han sido recogidos mediante scrapping a la página web de júpiter juegos, una tienda con local en sevilla enfocada en los juegos de mesa y en el merchandising.

Búsqueda por nombre

La búsqueda por nombre se realiza mediante whoosh. Dada una cadena de caracteres, devuelve el juego que contenga ese título. Si la palabra ingresada no se encuentra dentro del título del juego, no se devolverá ningún resultado.

Búsqueda por rango de precio

Dados dos floats, devuelve todos los juegos que se encuentren en ese rango. Muestra título y precio de cada juego.

Listado de puntuaciones

Muestra las primeras 1000 puntuaciones del dataset del sistema de recomendación. También muestra si esos juegos se encuentran en la tienda de júpiter juegos

Recomendación de juego

Dado un nombre de usuario, te recomienda juegos que posiblemente le puedan gustar a ese usuario. Esto se hace mediante un filtrado de recomendación colaborativo basado en usuarios.

Recomendación de un juego similar

Dado un ID de un juego de mesa, recomienda juegos similares al dado. Esto se hace mediante un filtrado de recomendación colaborativo basado en items.

Carga base de datos

La carga de la base de datos no debe ser utilizada a no ser que sea estrictamente necesario, ya que carga 14 millones de datos del dataset de recomendación, con lo cual puede tardar un tiempo elevado en ejecutarse.

La carga ejecuta 3 funciones: PopulateDB(), crear_schema() y populatePuntuacion()

PopulateDB carga la base de datos del scrapping realizado a júpiter juegos. Para ello, accede al catálogo de juegos de mesa, va accediendo uno a uno, comprueba que la url contenga juego de mesa, y comprueba que contenga todos los datos. Una vez ha hecho las comprobaciones correspondientes, carga las 10 primeras páginas en la base de datos.

crear_schema genera el esquema de whoosh para la posterior búsqueda de datos. En este esquema se introducen exactamente los mismos datos que los que contiene el modelo de JuegoDeMesa.

populatePuntuacion carga el dataset del sistema de recomendación. Se hace en un único modelo debido a que en principio no requiero de utilizar los otros datasets, y ya que la carga es lenta, opté por cargar un único modelo. En este se comprueba que el tamaño de lo recogido sea igual a 6, para evitar posibles problemas con el separador, y guarda usuario, puntuación, id del juego y nombre del juego.

Carga matriz

Para cargar el diccionario, simplemente se genera un diccionario de usuarios, en el que cada usuario tiene a su vez asociado un diccionario con juegos y con su puntuación a cada juego. Para la transformada, se intercambian los usuarios por los juegos.
